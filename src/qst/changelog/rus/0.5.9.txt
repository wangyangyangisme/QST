0.5.8.1 to 0.5.9 (pre-alpha) Changes

--====== Внимание! ======--

Данная версия QST часточно несовместима с предыдущими. Важные изменения коснулись очередности полей при генерации запроса. Теперь поля следуют друг за другом так, как они заданы в классе-хэндлере.

--=======================--

-= Рефакторинг QstField, QstBatch, QstAbstractModelHandler =-
- QstField теперь не принимает параметр FieldPurposes. Для каждого поля теперь будет одно назначение.
- QstBatch больше не сортирует поля по видимости. Поля идут так, как они заданы в генерирующей функции _selector().
- Добавлена роль поля RoleTreeLevel. Для правильной работы древовидной модели в хэндлере требуется указать поле с ролью RoleTreeLevel, обозначающее уровень элемента в дереве.

-= QstAbstractModelHandler =-
- Изменены интерфейсы функций viewKeyValue(), viewFieldValue(), comboBoxKeyValue(), fieldValue(), fieldsValueMap() etc. Теперь вместо необязательного параметра row нужно передавать QModelIndex.
- Исправлен баг с выборкой данных в текущей строке во view, если модель - древовидная.
- Незначительно обновлена документация, исправлены небольшие ошибки.

-= QstTreeQueryModel =-
- Модель переписана.
- Добавлена функция setLevelField(int).
- Разрешена функция setQuery(QSqlQuery).
- Добавлены виртуальные функции  _setupTree(), _columnTitles(QSqlRecord), _itemDataMap(QSqlRecord), _itemFlags(QSqlRecord).
